# Auto Import AI Assistant

Auto Import AI Assistant — MVP-инструмент для менеджера компании по импорту премиальных автомобилей.

Он помогает быстро обработать входящий запрос клиента: понять бюджет и пожелания, отобрать подходящие автомобили из CSV, подсветить риски и подготовить готовое сообщение для клиента.

## Какую проблему решает

Менеджер по автоимпорту часто получает свободные формулировки: «нужен семейный кроссовер до 8 млн, желательно Корея или Европа, Китай не рассматриваю». Такие запросы нужно быстро перевести в понятные критерии, сверить с текущими предложениями, отсеять неподходящие машины и объяснить клиенту выбор.

Проект показывает рабочий бизнес-пайплайн: от текста клиента до топа автомобилей, AI-анализа и сохранения лида.

## Как работает пайплайн

1. Пользователь вводит запрос клиента и загружает CSV с автомобилями.
2. LLM через Groq извлекает структурированные требования в строгий JSON.
3. Python фильтрует и ранжирует автомобили: бюджет, кузов, страна, бренд, год, пробег и приоритеты.
4. LLM получает только исходный запрос, извлеченные требования и top cars из CSV.
5. На выходе менеджер получает summary, топ-3 варианта, риски, вопросы клиенту и сообщение для Telegram.
6. Заявка сохраняется в `outputs/leads_log.csv`.

## Главный принцип

Числовые ограничения и фильтрация реализованы в Python, а LLM используется для понимания свободного текста, анализа плюсов/минусов и генерации коммуникации. Это снижает риск галлюцинаций и делает инструмент ближе к реальному бизнес-процессу.

LLM не принимает решение сама: она не придумывает рынок, цены, годы, пробеги и комплектации. Все числовые данные берутся из CSV, а модель объясняет уже отобранные варианты.

## Демо-данные

В `data/sample_cars.csv` лежит демонстрационная база из 35 автомобилей из Кореи, Китая и Европы. Цены и характеристики выглядят реалистично, но это не коммерческое предложение.

Для production нужно подключать актуальные источники данных, реальные курсы валют, пошлины, логистику, CRM и проверку доступности конкретных автомобилей.

## Запуск

Linux/macOS:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
streamlit run app.py
```

Windows:

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
streamlit run app.py
```

## Настройка `.env`

Создайте `.env` из примера:

```bash
copy .env.example .env
```

Добавьте ключ:

```env
GROQ_API_KEY=your_groq_api_key
GROQ_MODEL_MAIN=llama-3.3-70b-versatile
GROQ_MODEL_FAST=llama-3.1-8b-instant
```

Groq API key можно получить в консоли Groq: https://console.groq.com/

Если ключ не задан, приложение не падает: требования и анализ вернут безопасное сообщение о недоступности LLM.

## Как протестировать demo-запрос

1. Запустите `streamlit run app.py`.
2. В боковой панели оставьте включенным `Использовать demo CSV`.
3. Выберите один из demo-запросов и нажмите `Подставить demo-запрос`.
4. Нажмите `Проанализировать запрос`.
5. Проверьте извлеченные требования, таблицу автомобилей, AI-анализ и кнопку скачивания Markdown.

## Тесты

```bash
pytest
```

Тесты проверяют:

- отсечение автомобилей выше бюджета больше чем на 7%;
- влияние кузова и preferred brand на score;
- исключение страны;
- fallback-парсер JSON для ответа LLM.

## Ограничения MVP

- В проекте используются демонстрационные данные.
- Цены не являются коммерческим предложением.
- Для production нужно подключить реальные источники данных, CRM, курсы валют, расчет пошлин и логистики.
- LLM не должна использоваться как единственный источник истины по рынку.
- Нет авторизации, Telegram-интеграции, платежей и production-инфраструктуры.

## Что показать работодателю

Покажите, что проект не просто вызывает LLM, а разделяет ответственность:

- LLM понимает текст клиента и готовит объяснение;
- Python считает ограничения, score и ранжирование;
- CSV остается источником фактов;
- менеджер получает результат, который можно проверить и отправить клиенту.
